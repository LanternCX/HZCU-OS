   因为父进程先创建第一个子进程，然后父进程自己又继续创建第二个子进程，所以最终会有三个执行分支分别输出内容。

## Task 9：完成最后的编程题

### 执行命令

先复制一个参考文件：

```bash
cp fork7.c final_lab03.c
```

然后把 `final_lab03.c` 改成下面这份内容：

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/types.h>
#include <sys/wait.h>

int main(void)
{
    pid_t pid;
    int i;

    pid = fork();
    if (pid < 0) {
        perror("fork failed");
        return 1;
    }

    if (pid == 0) {
        for (i = 0; i < 5; i++) {
            printf("child: pid=%d ppid=%d\n", getpid(), getppid());
            system("/bin/ps");
            sleep(3);
        }
        exit(0);
    } else {
        for (i = 0; i < 5; i++) {
            printf("parent: pid=%d ppid=%d\n", getpid(), getppid());
            system("/bin/ps");
            sleep(3);
        }
        wait(NULL);
    }

    return 0;
}
```

保存后执行：

```bash
gcc final_lab03.c -o final_lab03
./final_lab03
```

### 本 task 要回答的问题

1. 程序功能是什么？
   程序先创建一个子进程，然后父子进程分别输出自己的编号和父编号，并且每隔 3 秒打印一次系统进程信息，共重复 5 次。最后由父进程等待子进程结束后再退出。
2. 程序是怎样完成要求的？
   先创建子进程，再在父子两个执行分支中分别进行循环输出。每轮输出当前进程信息和系统进程信息，然后暂停 3 秒。循环结束后，子进程直接结束，父进程等待子进程结束后再退出。
3. 运行结果说明了什么？
   可以看到父子两个执行分支都按要求重复输出多次，并且父进程在子进程结束后才最终结束。
