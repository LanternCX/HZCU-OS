4. 再改 `HandleData1`。
   把它开头等待的控制量，改成等待第一套控制量。
   把它计算加法时读取的数据，改成只读取第一套数据区。
   不要再从原来那组共用 `stack` 里取值。
5. 再改 `HandleData2`。
   把它开头等待的控制量，改成等待第二套控制量。
   把它计算乘法时读取的数据，改成只读取第二套数据区。
   不要再从原来那组共用 `stack` 里取值。
6. 最后改 `main`。
   把原来只初始化一次的 `sem_init(&sem,0,0);` 改成分别初始化两套控制量。
   线程创建顺序可以不变，但最终关系必须变成：
   `ReadData1` 只对应 `HandleData1`。
   `ReadData2` 只对应 `HandleData2`。
7. 改完后自己检查一遍，确认文件里已经不存在“两个读取线程还在共用同一组数据区和同一个控制量”的情况。

直接可用的改后代码如下：

```c
#include <stdio.h>
#include <pthread.h>
#include <semaphore.h>

#define MAXSTACK 100

int stack1[MAXSTACK];
int stack2[MAXSTACK];
int size1 = 0;
int size2 = 0;
sem_t sem1;
sem_t sem2;

void ReadData1(void)
{
    printf("ReadData1 begin:**********************\n");
    FILE *fp = fopen("1.dat", "r");
    while (!feof(fp)) {
        fscanf(fp, "%d %d", &stack1[0], &stack1[1]);
        sem_getvalue(&sem1, &size1);
        printf("ReadData1 stack1[0]=%d stack1[1]=%d sem1.value=%d\n", stack1[0], stack1[1], size1);
        sem_post(&sem1);
    }
    fclose(fp);
}

void ReadData2(void)
{
    printf("ReadData2 begin:**********************\n");
    FILE *fp = fopen("2.dat", "r");
    while (!feof(fp)) {
        fscanf(fp, "%d %d", &stack2[0], &stack2[1]);
        sem_getvalue(&sem2, &size2);
        printf("ReadData2 stack2[0]=%d stack2[1]=%d sem2.value=%d\n", stack2[0], stack2[1], size2);
        sem_post(&sem2);
    }
    fclose(fp);
}

void HandleData1(void)
{
    printf("Plus begin:**********************\n");
    while (1) {
        sem_getvalue(&sem1, &size1);
        sem_wait(&sem1);
        printf("Plus:%d+%d=%d sem1.value=%d\n", stack1[0], stack1[1], stack1[0] + stack1[1], size1);
    }
}

void HandleData2(void)
{
    printf("Multiply begin:**********************\n");
    while (1) {
        sem_getvalue(&sem2, &size2);
        sem_wait(&sem2);
        printf("Multiply:%d*%d=%d sem2.value=%d\n", stack2[0], stack2[1], stack2[0] * stack2[1], size2);
    }
}

int main(void)
{
    pthread_t t1, t2, t3, t4;
    sem_init(&sem1, 0, 0);
    sem_init(&sem2, 0, 0);
    pthread_create(&t1, NULL, (void *)HandleData1, NULL);
    pthread_create(&t2, NULL, (void *)HandleData2, NULL);
    pthread_create(&t3, NULL, (void *)ReadData1, NULL);
    pthread_create(&t4, NULL, (void *)ReadData2, NULL);
    getchar();
}
```

### 步骤 8：回答问题

1. 理解信号量的初始化操作、P 操作和 V 操作。
   这里 `sem_init(&sem,0,0)` 先把初始值设成 0，处理线程一开始不能直接继续，而要等读取线程先把数据准备好。读取线程每读一次就释放一次，处理线程每处理一次就先等待一次，这样先后顺序就被串起来了。
2. 分析出四个线程的前驱后继关系。
   题目要求先读取、后处理，因此 `ReadData1` 和 `ReadData2` 是前驱动作，`HandleData1` 和 `HandleData2` 是后继动作。也就是说，先有读取，后有处理；没有前面的数据，后面的处理就不能继续。
3. 若要求 `ReadData1` 读出的数据仅由 `HandleData1` 处理，`ReadData2` 读出的数据仅由 `HandleData2` 处理，请修改例程。
   当前程序把两路数据都放在同一份共享位置里，后面的处理关系是混在一起的。要满足题目要求，就要把数据区和控制关系拆成两套，让第一路只对应第一路，第二路只对应第二路。
