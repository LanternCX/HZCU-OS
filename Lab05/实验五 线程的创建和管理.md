# 实验五 线程的创建和管理

## 实验目的

1. 理解进程与线程的区别。
2. 实现单线程与多线程控制。
3. 了解 POSIX 线程机制，使用常用的线程操作函数。

## 实验内容

1. 观察单线程程序和多线程程序的区别。
2. 使用常用的线程操作函数。
3. 使用线程互斥锁。

## 实验步骤

### （一）基本概念

1. Linux 并不存在真正意义上的独立线程结构，线程通常被看作轻量级进程。创建线程所需资源更少，执行效率更高。
2. Linux 下多线程通常遵循 POSIX 线程接口，也就是 `pthread`。编写多线程程序时需要包含头文件 `pthread.h`，编译时通常要加 `-lpthread`。

### （二）本实验涉及的常用函数

#### 1. 创建新线程

```c
int pthread_create(pthread_t *thread, const pthread_attr_t *attr,
                   void *(*start_routine)(void *), void *arg);
```

函数说明：

`pthread_create()` 用于创建新线程。线程创建成功后，会从 `start_routine` 指定的位置开始运行。

参数说明：

1. `thread`：返回线程标识。
2. `attr`：线程属性，`NULL` 表示默认属性。
3. `start_routine`：线程执行函数入口。
4. `arg`：传给线程函数的参数。

返回值：

1. 成功返回 `0`。
2. 失败返回错误码。

头文件：

```c
#include <pthread.h>
```

#### 2. 线程终止

```c
void pthread_exit(void *retval);
```

函数说明：

线程调用 `pthread_exit()` 后结束执行，并可返回一个结果指针，通常可写为 `NULL`。

#### 3. 线程等待

```c
int pthread_join(pthread_t thread, void **retval);
```

函数说明：

`pthread_join()` 以阻塞方式等待指定线程结束。等待结束后，被等待线程的资源会被回收。

参数说明：

1. `thread`：目标线程标识。
2. `retval`：用于接收线程返回值。

返回值：

1. 成功返回 `0`。
2. 失败返回错误号。

#### 4. 互斥锁初始化

静态互斥锁可使用 `PTHREAD_MUTEX_INITIALIZER` 初始化；动态互斥锁可使用下列函数：

```c
int pthread_mutex_init(pthread_mutex_t *restrict mutex,
                       const pthread_mutexattr_t *restrict attr);
```

#### 5. 锁定互斥锁

```c
int pthread_mutex_lock(pthread_mutex_t *mutex);
```

函数说明：

调用成功后，当前线程获得互斥锁；如果锁已被其他线程占用，则当前线程会等待。

#### 6. 释放互斥锁

```c
int pthread_mutex_unlock(pthread_mutex_t *mutex);
```

### （三）使用线程的例程

#### 1. 单线程 hello world 程序

```c
#include <stdio.h>

const NUM = 5;

int main()
{
    p_msg("hello");
    p_msg("world\n");
}

void p_msg(char *m)
{
    int i;
    for (i = 0; i < NUM; i++) {
        printf("%s", m);
        fflush(stdout);
        sleep(1);
    }
}
```

编译运行后，观察输出并思考：

1. 该程序是顺序执行的，先完成一次函数调用，再进行下一次函数调用。
2. `fflush()` 的作用是什么。

#### 2. 多线程 hello world 程序

```c
#include <stdio.h>
#include <pthread.h>

main()
{
    pthread_t t1, t2;
    void *p_msg(char *);

    pthread_create(&t1, NULL, p_msg, (void *)"hello");
    pthread_create(&t2, NULL, p_msg, (void *)"world\n");
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
}

void *p_msg(char *m)
{
    int i;
    for (i = 0; i < 5; i++) {
        printf("%s", m);
        fflush(stdout);
        sleep(1);
    }
    return NULL;
}
```

编译运行后，观察输出并思考：

1. 从输出内容观察线程并发执行现象。
2. 使用 `ps -aL`、`ps -efL`、`pstree -p` 观察线程信息。
3. 理解 `pthread_create()` 的作用。
4. 理解 `pthread_join()` 的作用，并尝试删除后比较结果变化。
5. 改写代码，使相同输出由父子进程并发完成，体会线程和进程的差异。

#### 3. 观察线程编号和执行次序

```c
#include <stdio.h>
#include <stdlib.h>
#include <pthread.h>
#include <sys/syscall.h>

void *PrintThread(void *);
#define Num_Threads 3

int main()
{
    int i, ret;
    pthread_t a_thread;
    int ThdNum[Num_Threads];

    for (i = 0; i < Num_Threads; i++)
        ThdNum[i] = i;

    for (i = 0; i < Num_Threads; i++) {
        ret = pthread_create(&a_thread, NULL, PrintThread, (void *)&ThdNum[i]);
        if (ret == 0)
            printf("Thread launched succeddfully\n");
    }

    i = getchar();
    return 0;
}

void *PrintThread(void *num)
{
    int i;
    for (i = 0; i < 3; i++) {
        printf("thread number is %d,my pid = %d,my lwp id = %d,my tid = %u\n",
               *((int *)num), getpid(), syscall(SYS_gettid), pthread_self());
        sleep(1);
    }
    return NULL;
}
```

编译运行后，观察输出并思考：

1. 输出顺序为什么不固定。
2. 删除 `sleep(1)` 后，输出会怎样变化。
3. 各种 ID 分别表示什么。

#### 4. 使用互斥锁

```c
#include <stdlib.h>
#include <stdio.h>
#include <pthread.h>
#include <errno.h>
#include <sys/types.h>
#include <sys/syscall.h>

int gnum = 0;
pthread_mutex_t mutex;

static void pthread_add2(void);
static void pthread_add3(void);

pid_t gettid()
{
    return syscall(SYS_gettid);
}

int main(void)
{
    pthread_t pt_1 = 0;
    pthread_t pt_2 = 0;

    printf("main programme start,pid = %d,num = %d!\n", getpid(), gnum);
    pthread_mutex_init(&mutex, NULL);

    pthread_create(&pt_1, NULL, (void *)pthread_add2, NULL);
    pthread_create(&pt_2, NULL, (void *)pthread_add3, NULL);

    pthread_join(pt_1, NULL);
    pthread_join(pt_2, NULL);
    printf("main programme exit!\n");
    return 0;
}

static void pthread_add2(void)
{
    int i = 0;
    printf("This is pthread_1!PID = %d,LWP = %d,tid = %lu\n",
           getpid(), gettid(), pthread_self());
    for (i = 0; i < 3; i++) {
        // pthread_mutex_lock(&mutex);
        gnum++; sleep(1); gnum++;
        printf("Thread_1 add 2 to num:%d\n", gnum);
        // pthread_mutex_unlock(&mutex);
    }
    pthread_exit(NULL);
}

static void pthread_add3(void)
{
    int i = 0;
    printf("This is pthread_2!PID = %d,LWP = %d,tid = %lu\n",
           getpid(), gettid(), pthread_self());
    for (i = 0; i < 4; i++) {
        // pthread_mutex_lock(&mutex);
        gnum++; sleep(1); gnum++; sleep(1); gnum++;
        printf("Thread_2 add 3 to num:%d\n", gnum);
        // pthread_mutex_unlock(&mutex);
    }
    pthread_exit(0);
}
```

编译运行后，观察输出并思考：

1. 保留注释时，为什么输出会混乱。
2. 删除 `sleep()` 后，混乱现象为什么会减轻，但这是否算真正解决问题。
3. 恢复 `sleep()` 并启用互斥锁后，输出有何变化。
4. 掌握互斥锁的初始化、加锁和解锁。
5. 比较“删除 `sleep()`”和“启用互斥锁”两种做法的差异。
6. 尝试为例程 3 加互斥锁，使输出不交错。

#### 5. 练习题：教材习题 4.9

```c
#include <stdio.h>
#include <stdlib.h>
#include <pthread.h>
#include <errno.h>
#include <unistd.h>

int myglobal = 10;

void *thread_function(void *arg)
{
    int i, j;
    for (i = 0; i < 20; i++) {
        j = myglobal;
        j++;
        printf(".");
        fflush(stdout);
        usleep(1000);
        myglobal = j;
    }
    return NULL;
}

int main()
{
    pthread_t mythread;
    int i;

    if (pthread_create(&mythread, NULL, thread_function, NULL)) {
        printf("error creating thread!\n");
        abort();
    }

    for (i = 0; i < 20; i++) {
        myglobal++;
        printf("O");
        fflush(stdout);
        usleep(1000);
    }

    if (pthread_join(mythread, NULL)) {
        printf("error joining thread!\n");
        abort();
    }

    printf("\nmyglobal equald %d\n", myglobal);
    exit(0);
}
```

要求：

1. 观察原程序为什么得不到期望值。
2. 参照前一个例程加入互斥锁，使结果稳定达到期望值。
